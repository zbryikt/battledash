徵件評選系統平台之頁面、元件與互動等介面設計參考

介面概觀

依功能面向區分，徵件評選系統平台分為以下五個子系統：

  • 評選系統 - 統整評選相關事項。
  • 管理系統 - 控制活動資訊、權限等設定。
  • 提案系統 - 瀏覽、建立、編輯與管理提案。
  • 用戶系統 - 用戶資訊管理與訊息通知。
  • 頁面系統 - 設計並瀏覽活動各項資訊的公開頁面。

評選系統

評選子系統統整評選相關事項，提供必要之元件以利評審、初審員與工作人員進行評選作業之協作。由於資訊較多，評選子系統多採試算表式表格設計，並採用類似 Google Spreadsheet 的即時存檔設計，不需另外存檔，以降低系統操作之複雜度。

同時，在評選會議時有多人討論跟即時修改的需求，因此實作時應支援即時多人協作，表單 ( 如評審總表 ) 可同時多人編輯，個人表之編輯亦會即時回饋至總表。

個人初審表

個人初審表主要用於提案初審階段。初審表列出所有提案，同時將提案內容並列供審核員參考。

  • 點擊提案名稱後，提案內容呈現於右方面板。
  • 依照初審資格針對各個提案逐一審核，直接點擊狀態按鈕變更審核結果。
  • 若有爭議或難以判別之項目，可跳過審核並在評論處提供細節資訊。
  • 最左方匯整各別提案的審核通過狀況以及評論狀況。
  • 點擊各個項目名稱欄可依各別欄位排序。
  • 點擊審核條件連結彈出初審條件的細節描述，供審核員參考。

個人評選表

個人評選表為各個評審專用的評分表。評選表列出初審通過之提案，並依給分標準事先畫出指定欄位供評審使用。

  • 點擊提案名稱後，提案內容呈現於右方面板。
  • 評審依照給分標準針對各個提案逐一評分，總分與排名依評分自動更新。
  • 短評欄位依需求可做為評審加註使用，或者做為主辦單位參考用。
  • 初審不通過之提案預設不顯示，但可點擊按鈕展開。
  • 點擊各個項目名稱欄可依各別欄位排序。
  • 以進度條提示評審剩餘進度，並可在完成時提交結果通知工作人員。

評選總表

評選總表為匯整各評審評分後的總表，提供各提案的整體評價，主要用於決定獲選名單使用。

  • 點擊提案名稱後，彈出該提案的評分細節，包含各初審員的評論、評審的給分以及評論。
  • 各評審給分自動匯入，並自動計算各人排名與整體排名。
  • 各別提案之最高 / 最低分評審以特別底色標示，以協助討論。
  • 初審不通過之提案預設不顯示，但可點擊按鈕展開。
  • 點擊各個項目名稱欄可依各別欄位排序。
  • 點擊初審欄可查看初審員的評論。
  • 評審確定決選名單後，在獲選欄位點擊註記。
